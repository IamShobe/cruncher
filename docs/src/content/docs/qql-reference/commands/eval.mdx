  name="eval"

  syntax="| eval newField = expression"

## Parameters

- `newField = expression`: Assigns the result of the expression to a new or existing field.

## Supported Functions and Expressions

You can use arithmetic, logical, and conditional expressions, as well as functions, in `eval` assignments. Common supported functions include:

- Mathematical: `+`, `-`, `*`, `/`
- String: [`lower(str)`](/qql-reference/functions/strings/lower), [`upper(str)`](/qql-reference/functions/strings/upper), [`length(str)`](/qql-reference/functions/strings/length), [`trim(str)`](/qql-reference/functions/strings/trim)
- Number: [`abs(x)`](/qql-reference/functions/numbers/abs), [`round(x)`](/qql-reference/functions/numbers/round), [`ceil(x)`](/qql-reference/functions/numbers/ceil), [`floor(x)`](/qql-reference/functions/numbers/floor)
- Boolean: [`contains(str, substr)`](/qql-reference/functions/booleans/contains), [`startsWith(str, prefix)`](/qql-reference/functions/booleans/startsWith), [`endsWith(str, suffix)`](/qql-reference/functions/booleans/endsWith), [`match(str, regex)`](/qql-reference/functions/booleans/match), [`isNull(x)`](/qql-reference/functions/booleans/isNull), [`isNotNull(x)`](/qql-reference/functions/booleans/isNotNull)
- Conditional: [`if(condition, then, else)`](/qql-reference/functions/conditional/if), [`case((cond1, val1), ..., elseVal)`](/qql-reference/functions/conditional/case)

> Only functions supported by your Cruncher deployment are available. See the main documentation for a full list.

## Usage

- Use `eval` to compute new values, transform fields, or perform calculations.
- You can assign multiple fields by chaining multiple `eval` commands in your pipeline.
- Use conditional logic to create new fields based on complex criteria.

## Examples

<QQLCode code={`| eval duration_sec = duration / 1000`} />

This creates a `duration_sec` field by dividing `duration` by 1000.

<QQLCode code={`| eval is_error = (status == "error")`} />

This creates a boolean field `is_error` that is `true` when `status` equals `"error"`.

<QQLCode code={`| eval user_lower = lower(user)`} />

This creates a new field `user_lower` with the lowercase value of `user`.

<QQLCode code={`| eval error_type = if(status == "error", "critical", "normal")`} />

This creates a new field `error_type` based on the value of `status`.

<QQLCode code={'| eval match_found = match(message, `^Error:.*`)'} />

This creates a boolean field `match_found` that is `true` when `message` matches the regex.

<QQLCode code={`| eval group = case(status == "error", "A", status == "warn", "B", "C")`} />

This creates a field `group` with value `"A"` for errors, `"B"` for warnings, and `"C"` otherwise.
