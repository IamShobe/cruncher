The `case` function evaluates a series of conditions and returns the value associated with the first condition that is true. If none match, it returns the default value.

## Signature

```
case(condition1, value1, condition2, value2, ..., default_value) → any
```

## Parameters

| Name | Type | Description |
|------|------|-------------|
| `condition1`, `condition2`, ... | boolean | Conditions evaluated in order |
| `value1`, `value2`, ... | any | Values returned if the corresponding condition is `true` |
| `default_value` | any | Value returned if none of the conditions are `true` (last argument) |

## Returns

`any` — Returns the value paired with the first `true` condition. If no conditions are true, returns `default_value`.

## Edge Cases

- If a `condition` is `null`, it is treated as `false` (evaluation continues)
- Evaluation stops at the first `true` condition (short-circuit behavior)
- The `default_value` (last argument) is required
- Value types can vary across branches; return type depends on matched value
- Odd number of arguments required: pairs of (condition, value) plus the default

## Examples

### Simple Status Mapping

```qql
eval status_label = case(status == "error", "Error", status == "warn", "Warning", "OK")
```

Maps status codes to readable labels: "Error", "Warning", or "OK".

### Log Level Categorization

```qql
eval severity_tier = case(
  level == "FATAL", 1,
  level == "ERROR", 2,
  level == "WARN", 3,
  level == "INFO", 4,
  5
)
```

Assigns severity tier numbers: FATAL=1, ERROR=2, WARN=3, INFO=4, other=5.

### Response Time Classification

```qql
eval performance = case(
  response_time_ms < 100, "fast",
  response_time_ms < 500, "normal",
  response_time_ms < 1000, "slow",
  "very slow"
)
```

Categorizes requests by response time.

### Complex Business Logic

```qql
eval customer_segment = case(
  total_spent > 10000, "premium",
  total_spent > 1000, "standard",
  login_count == 0, "inactive",
  "active"
)
```

Multi-criteria customer segmentation.

### Environment-Based Configuration

```qql
eval should_alert = case(
  environment == "production" AND error_count > 10, true,
  environment == "staging" AND error_count > 50, true,
  false
)
```

Alert conditions vary by environment.
